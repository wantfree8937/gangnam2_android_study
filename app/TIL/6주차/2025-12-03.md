# Jetpack Compose 요약

### 1️⃣ 기본 원칙 (Fundamentals)
* **함수 명명 규칙 (PascalCase)**
    * `@Composable` 어노테이션이 붙은 함수는 **대문자**로 시작해야 합니다.
    * *(명사형 이름을 권장합니다.)*
    * `ex) fun Greeting() { ... }`
* **Stateless 지향**
    * 대부분의 UI 컴포저블은 **Stateless(상태를 가지지 않음)**로 설계합니다.
    * 내부에서 변수를 직접 들고 있기보다는, 외부에서 **파라미터**로 데이터를 받아 그리기만 하는 것이 좋습니다.
* **변수 사용 주의**
    * 컴포저블 함수 내부의 일반 변수는 다시 그려질 때(Recomposition) **초기화**됩니다.
    * 상태가 필요하다면 `remember { mutableStateOf(...) }`를 사용해야 합니다.

### 2️⃣ 데이터와 렌더링 (Data Flow)
* **단방향 데이터 흐름 (Unidirectional Data Flow)**
    * **Data Down:** 데이터는 파라미터를 통해 **위에서 아래로** 흐릅니다. (UI 갱신)
    * **Event Up:** 이벤트는 콜백 함수(Lambda)를 통해 **아래에서 위로** 전달합니다.
* **불변 객체 권장 (Immutability)**
    * **규칙:** 파라미터로 `data class`를 받을 때, 내부 프로퍼티는 `val`(불변)로 선언하는 것이 좋습니다.
    * **이유:** `var`(가변)가 포함되어 있으면 Compose가 데이터 변경을 추적하기 어려워 무조건 다시 그립니다. 불변 객체는 데이터가 안 바뀌었으면 **그리기 건너뛰기(Smart Recomposition)**가 가능하여 성능이 향상됩니다.

### 3️⃣ UI 요소와 속성 (Components & Attributes)
* **단위 (Units)**
    * **Text Size:** `sp` (사용자 폰트 설정에 따라 배율 적용)
    * **Layout Size:** `dp` (화면 밀도에 독립적)
* **주요 컴포저블**
    * **`Text()`:** 첫 번째 인자(`text`)는 필수이며 디폴트 값이 없습니다.
    * **`ElevatedButton()`:** 그림자가 있어 떠 있는 듯한 느낌을 주는 기본 버튼입니다.
* **Modifier (수식어)**
    * 모든 컴포저블의 **디자인, 배치, 동작**을 제어하는 핵심 속성입니다.
    * 크기, 배경색, 클릭 이벤트, 패딩 등을 설정합니다.

### 4️⃣ 개발 도구 및 리소스 (Tools & Resources)
* **@Preview**
    * 이 어노테이션을 붙이면 에뮬레이터 실행 없이 **Design 탭(오른쪽)**에서 UI를 미리 볼 수 있습니다.
* **리소스 명명 규칙 (Snake Case)**
    * 이미지(Drawable), 폰트(Font) 등의 파일명은 **소문자**와 **언더바(_)**만 사용해야 합니다.
    * ❌ `MyFont.ttf`, `bg-image.png` (대문자, 대시 불가)
    * ⭕ `my_font.ttf`, `bg_image.png`
